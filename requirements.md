# SSM 프로젝트 요구사양

## 1. 수식 편집기 (Formula Editor)

### 1.1 개요
하단 패널에 배치될 수식 편집기로, YAML에 정의된 파라미터를 활용한 사칙연산 수식을 작성하고 평가할 수 있는 기능을 제공합니다.

### 1.2 기능 요구사항

#### 1.2.1 수식 작성
- **지원 연산**: 사칙연산 (+, -, *, /)
- **변수 사용**: YAML에 정의된 파라미터 키만 유효한 변수로 사용 가능
- **괄호 지원**: 연산 우선순위 조절을 위한 괄호 사용 가능

#### 1.2.2 변수 자동완성
- IntelliJ 스타일의 자동완성 기능 제공
- 입력 중 사전 등록된 변수명 제안
- 키보드 탐색 지원 (↑, ↓, Enter)
- 변수명 부분 일치 검색

#### 1.2.3 수식 유효성 평가
- 실시간 구문 검사
- 미등록 변수 사용 시 오류 표시
- 구문 오류(syntax error) 감지
- 수식 평가 결과 실시간 표시

#### 1.2.4 결과 리포트
- 수식 평가 결과 표시
- 사용된 변수와 현재 값 표시
- 오류 메시지 표시 (오류 발생 시)
- 수식 계산 과정 표시 (옵션)

### 1.3 UI/UX 요구사항
- 깔끔하고 직관적인 인터페이스
- 실시간 피드백 제공
- 색상 코드: 
  - 유효한 수식: 초록색 표시
  - 오류가 있는 수식: 빨간색 표시
  - 변수: 파란색 강조
- 자동완성 드롭다운: 현재 커서 위치에 표시

### 1.4 기술 사양
- React 기반 컴포넌트
- 실시간 유효성 검사
- 키보드 이벤트 핸들링
- JavaScript `eval()` 대신 안전한 수식 파싱 사용

### 1.5 향후 확장 계획
- 수학 함수 지원 (sin, cos, sqrt 등)
- 수식 히스토리 저장
- 수식 템플릿 기능
- 복잡한 조건문 지원
