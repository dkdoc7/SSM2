# 디자인 가이드라인 (Design Guidelines)

이 문서는 프로젝트의 핵심 디자인 원칙과 UI 스타일을 기록하여 일관성을 유지하기 위해 작성되었습니다.

## 1. 핵심 컨셉
- **White Tone & Minimalist**: 밝고 깨끗한 화이트 톤의 라이트 모드 테마.
- **Flat Layout**: 불필요한 테두리, 배경 박스, 구분선을 최소화한 평평하고 간결한 배치.

## 2. 색상 팔레트 (Color Palette)
- `Background`: `#f8f9fa` (연한 그레이 배경)
- `Card/Input`: `#ffffff` (순백색)
- `Text Primary`: `#2c3e50`
- `Accent Color`: `#3498db` (블루 계열 액센트)
- `Border Color`: `#dee2e6`

## 3. UI 컴포넌트 스타일
### 파라미터 행 (Parameter Row)
- **배치**: `[라벨 - 입력 필드 - 단위 - 정보 아이콘]` 순서의 한 줄 배치.
- **명명 규칙 (Naming Rule)**: 파라미터 이름이 언더바(`_`)로 시작하는 경우 (예: `_HiddenParam`), UI에서 **라벨은 보이지 않도록** 처리함. 이 경우 입력 필드가 행의 왼쪽부터 시작됨.
- **조건부 렌더링 (Conditional Rendering)**: `Condition` 속성을 통해 특정 조건이 충족될 때만 파라미터를 표시할 수 있음. (예: `Condition: _Frame=="Primary"`) JavaScript 논리 연산식을 지원함.
- **수식 평가 (Expression Evaluation)**: `Default` 값에 괄호로 감싼 수식을 작성하면 자동으로 계산됨. (예: `Default: (A_Volt + B_Volt)`). 파라미터와 연산자 사이는 공백으로 구분. 수식이 유효하지 않거나 결과가 범위를 벗어나면 tomato 색상으로 경고 표시하고, 입력 필드 아래에 구체적인 오류 메시지 표시 (예: "⚠️ 수식 오류: A_Volt is not defined").
- **간격**: 행 사이의 구분선은 없으며, `0.4rem` 수준의 수직 패딩으로 간격 유지.
- **정렬**: 
  - 라벨: 왼쪽 정렬 (min-width: 120px)
  - 입력값: **오른쪽 정렬** (수치 가독성 강조)
  - 에디터(YAML): 왼쪽 정렬 (Monospace 코드 폰트)

### 입력 필드 (Input Fields)
- **Radius**: `6px`
- **Focus**: `#3498db` 색상의 테두리와 미세한 글로우 효과.
- **Spin Buttons**: 숫자 입력창의 업다운 버튼은 숫자와 `12px` 이상의 간격 유지.

### Group 타입 (Segmented Control)
- **형태**: 양쪽 끝이 둥근 캡슐 모양의 바(Bar).
- **스타일**: 컨테이너는 `#3498db` 배경이며, 선택된 아이템은 흰색 배경에 파란색 굵은 글씨로 표시됨. 선택되지 않은 아이템은 파란색 배경 위에 흰색 글씨로 표시됨.
- **높이**: `36px`, 내부 여백을 고려하여 부드러운 전환 효과 적용.

### 인포메이션 아이콘 & 툴팁
- **아이콘**: `17px` 크기의 작은 원형 `i` 아이콘.
- **툴팁**: `rgba(0, 0, 0, 0.75)` 배경의 다크 스타일 버블 툴팁. 하단 중앙에 말꼬리(tail) 포함.

## 4. 코드 구현 원칙
- 인라인 스타일보다는 프로젝트의 `--var` 변수를 최대한 활용할 것.
- 새로운 컴포넌트 추가 시에도 위 '한 줄 배치'와 '라이트 모드' 감성을 유지할 것.
